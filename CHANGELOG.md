# 更新日志

本项目遵循语义化版本号：`主版本.次版本.修订版本`。

## 未发布

### 修复

- 修复从悬浮球内点击“更新”后第一次安装可能只退出旧进程、新版未立即启动的问题；安装器现在会等待旧悬浮球释放单实例锁后再启动新版。

## v0.1.9 - 2026-06-16

### 新增

- 更新入口改为应用内下载新版安装器：用户确认下载后显示下载进度，下载完成后再确认是否立即安装。
- 下载失败时保留打开 GitHub Release 下载页面的兜底入口。
- 自动更新检查频率改为启动后约 15 秒检查一次，之后每 1 小时检查一次。
- 右键菜单新增置灰的当前版本项，方便确认正在运行的版本号。
- 折叠气泡里的额度重置时间改为更浅灰色，降低和主额度信息的视觉竞争。
- 双选紧凑折叠态改为固定列位排版，让 5小时/1周、剩余额度和重置时间的间隔更均匀。

## v0.1.8 - 2026-06-16

### 新增

- 显示窗口改为复选逻辑，可只看 5小时、只看 1周，或在折叠气泡中同时紧凑显示两项。
- 折叠气泡中，钟表图标表示额度重置时间，循环图标表示数据刷新时间；双选折叠态采用两行紧凑排版，刷新时间固定在右下角，避免切换时跳变。

### 修复

- 刷新数据时不再读取数据文件里的 `active_window` 覆盖用户显示偏好，避免用户选中 1周后点击刷新又跳回 5小时。
- 手动刷新会主动读取一次本机 Codex 会话快照，刷新时间显示数据快照时间，避免旧数据被误认为刚刷新过。

## v0.1.7 - 2026-06-16

### 新增

- 发现新版本时，悬浮球折叠态和展开态都会显示可点击的“更新”标记，托盘图标显示新版本提示，并在托盘菜单提供更新入口。
- 启动后会安静检查一次 GitHub 最新 Release，后续定期检查，不再只依赖用户手动右键检查。

### 修复

- 更新检查优先通过 GitHub Release latest 跳转解析最新版本，避免匿名 GitHub API 限流时误报“网络连接失败”。
- 构建、安装和卸载脚本移除 PowerShell 执行策略绕过参数、字符数组字符串拼接和隐藏临时自删除脚本，降低安全软件误报概率。
- 安装/卸载时如果当前环境不允许读取 `Win32_Process`，会跳过旧进程清理并继续执行，不再因为进程表读取被拒绝而直接失败。

## v0.1.6 - 2026-06-16

### 新增

- 新增系统托盘图标。双击托盘图标会把悬浮球找回到当前可用屏幕，并显示“悬浮球在这里”的短暂提示。
- 托盘右键菜单新增“定位悬浮球”和“退出”。

### 修复

- 悬浮球主窗口会从任务栏隐藏，避免显示 Python 默认图标。
- 新增可见性看门狗：分辨率变化、显示器拔插或重新登录后，如果悬浮球落到不可见区域，会自动夹回可用工作区。

## v0.1.5 - 2026-06-16

### 修复

- 合并 PR #1：修复部分多屏环境下 `HMONITOR` 句柄值过大导致 `GetMonitorInfoW` 参数转换溢出的问题。
- 避免主显示器工作区读取失败后，悬浮球被夹到其他屏幕，看起来像“启动后消失”。

## v0.1.4 - 2026-06-15

### 新增

- 安装后的目录新增 `卸载悬浮球.bat`，普通用户可以直接卸载应用文件。
- 开始菜单新增“卸载 Codex 额度悬浮球”快捷方式。
- 新增应用图标 `docs/assets/codex-bubble.ico`，用于安装器、快捷方式和悬浮球窗口。

### 调整

- 发布验证流程新增卸载烟测：安装器先安装到临时目录，再调用卸载脚本确认安装目录会被移除。
- 安装器构建时会把应用图标编入 `.exe`。

## v0.1.3 - 2026-06-15

### 新增

- Release 产物改为 Windows 安装器 `codex-bubble-setup-v0.1.3.exe`，普通用户不再需要下载源码 zip 后手动解压。
- 新增安装器内部脚本：安装到 `%LOCALAPPDATA%\Programs\CodexBubble`，创建桌面和开始菜单快捷方式，并在覆盖安装时退出旧版悬浮球和后台同步器。

### 调整

- GitHub Actions 改为在 tag 发布时现场构建安装器并上传，不再依赖仓库中提交的 zip 包。
- 程序内“检查更新”优先识别新版安装器下载地址，兼容旧版本 zip 资产作为兜底。
- 发布验证流程增加安装器和内部 payload 检查，避免运行时数据、日志、缓存进入安装包。
- `.gitignore` 增加发布产物和安装器临时文件规则，减少二进制产物误提交。

## v0.1.2 - 2026-06-15

### 新增

- 右键菜单新增“检查更新”，可联网读取 GitHub 最新 Release，并打开下载页面。
- 悬浮球启动时会自动确保后台同步器在运行，避免只启动前台时一直停留在静态数据。

### 修复

- 运行目录和 `%LOCALAPPDATA%` 都不可写时，配置、数据和日志会继续回退到系统临时目录，避免首次启动直接失败。
- 更新检查在后台线程执行，避免联网等待时卡住悬浮球界面。
- 悬浮球和后台同步器默认不允许多开，重复启动时新进程会自动退出。
- 右键菜单改用系统原生菜单，避免自绘菜单被悬浮窗透明层遮挡或定位到屏幕左上角。
- 优化拖动边界判断，拖动过程中不夹取位置，松手后按鼠标所在显示器工作区收回。
- 修复拖动时可能固定在左上角的问题：窗口移动统一改用 Tk `geometry`，避免 Win32 `SetWindowPos` 对 Tk 透明窗口失效。
- 在开发仓库中运行时，运行时配置不再写入 tracked 的默认 `config/` 文件。

## v0.1.1 - 2026-06-15

### 修复

- 修复安装目录不可写时启动失败的问题。
- 运行时配置、额度数据和日志现在会优先写入项目目录；如果项目目录不可写，会自动回退到 `%LOCALAPPDATA%\CodexBubble`。
- 后台同步器不再因为无法创建 `logs/` 目录而静默退出。

## v0.1.0 - 2026-06-15

第一个可公开下载的预览版。

### 新增

- Windows 桌面 Codex 额度悬浮球。
- 本机 Codex 会话快照读取：只解析 `rate_limits`，不读取 cookie、token 或 `auth.json`。
- 后台同步器：每 60 秒刷新一次本机额度数据。
- 中文启动入口：`启动悬浮球.bat`。
- 5 小时 / 1 周额度窗口切换。
- 未连接状态统一显示 `-`。
- 多显示器支持：可拖到副屏，支持负坐标屏幕布局，右键菜单限制在鼠标所在屏幕内。
- 正式项目结构：`src/`、`config/`、`scripts/`、`docs/`、`releases/`。
- Agent 开发文档：`AGENTS.md`。

### 安全边界

- 不读取浏览器 cookie。
- 不读取登录 token。
- 不读取 `auth.json`。
- 不调用远程额度接口。
- 不关闭、重启或控制 Codex 进程。
- 发布包不包含本机 `data/`、`logs/` 或真实额度数据。
