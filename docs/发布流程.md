# 发布流程

本项目用 Git tag + GitHub Release 做版本发布。

## 版本号规则

使用语义化版本号：

- `0.1.0`：第一个公开预览版。
- `0.x.0`：新增功能或明显体验改进。
- `0.x.y`：问题修复、文档修正或小范围兼容改动。
- `1.0.0`：功能和安装体验稳定后发布正式版。

## 发布前检查

1. 更新 `VERSION`。
2. 更新 `CHANGELOG.md`。
3. 本地构建并核验：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_release.ps1
```

4. 确认 `scripts\verify_release.ps1` 输出核验通过。
5. 提交代码和发布包。
6. 创建并推送版本 tag。

发布前必须确认发布包不包含：

- `.git/`
- `data/`
- `logs/`
- `__pycache__/`
- `*.log`
- 本机生成的 `codex_usage_data.json`

## GitHub Release 内容

Release 标题格式：

```text
Codex 额度悬浮球 vX.Y.Z
```

Release 附件：

```text
codex-bubble-vX.Y.Z.zip
```

Release 说明从 `CHANGELOG.md` 对应版本复制。

## 自动发布

仓库包含 `.github/workflows/release.yml`。推送 `v*` 标签后，GitHub Actions 会自动创建 Release，并上传对应的版本包：

```text
releases/codex-bubble-vX.Y.Z.zip
```

注意：没有本地核验通过，不得推送 tag 触发自动发布。
